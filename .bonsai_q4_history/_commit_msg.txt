Switched q4mv_spec_f16io_resadd (O proj, 28× per token) and q4mv_spec_f16in (LM head, 1× per token) from `constant half*` input to `device const half*` with cooperative threadgroup memory caching. The interleaved SIMD access pattern means different lanes read different input addresses, defeating the constant cache's broadcast optimization. TG memory (on-chip SRAM) provides single-cycle access for non-uniform reads. Each TG cooperatively loads K halfs (4KB for K=2048) from device memory, then all SIMD groups read from fast TG memory. Result: 220.0-220.2 tok/s (up from 214-218 baseline), consistent across 3 runs. Prefill also improved to ~105 tok/s. p99 latency stable at ~4750μs.
